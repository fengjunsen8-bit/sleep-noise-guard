# 睡眠噪音守卫中文说明

睡眠噪音守卫用于监听本地环境噪音。当噪音超过设定阈值，并满足“出现几秒后触发”或“出现几次后触发”的条件后，程序会播放反馈音效，并记录日志。

## 已支持功能

- 实时估算环境分贝
- 超阈值后触发反馈音效
- 可设置触发阈值、持续秒数、出现次数、冷却时间
- 可设置每次触发后反馈音频播放次数
- CSV 日志记录噪音出现时间、分贝、触发次数、反馈次数和音效名称
- 显示本小时和今日噪音/触发统计
- macOS 原生中文界面
- Windows 中文桌面界面源码和 EXE 自动构建配置
- Android 中文原生 APK 源码和自动构建配置

## macOS 打包为 DMG

在 macOS 上运行：

```bash
chmod +x scripts/build_macos_app.sh scripts/build_macos_dmg.sh
scripts/build_macos_dmg.sh
```

产物：

```text
outputs/睡眠噪音守卫.dmg
```

也可以直接打开 `.app`：

```bash
open "outputs/睡眠噪音守卫.app"
```

## Windows 打包为 EXE

Windows EXE 通过 GitHub Actions 的 `windows-latest` runner 构建。工作流文件：

```text
.github/workflows/build-release.yml
```

也可以在 Windows 本地构建：

```powershell
python -m pip install --upgrade pip
python -m pip install -r requirements-build.txt
pyinstaller packaging/windows/sleep-noise-guard-windows.spec --noconfirm
```

产物：

```text
dist/睡眠噪音守卫.exe
```

## Android 打包为 APK

Android 项目目录：

```text
android/
```

GitHub Actions 会执行：

```bash
cd android
gradle assembleDebug
```

产物：

```text
android/app/build/outputs/apk/debug/app-debug.apk
```

Android 版本会请求麦克风权限，使用手机麦克风估算分贝，并用系统提示音作为反馈音。

## GitHub 自动构建

推送到 GitHub 后，可以在 Actions 页面手动运行：

```text
Build Release Artifacts
```

它会生成三个 artifact：

- `sleep-noise-guard-macos-dmg`
- `sleep-noise-guard-windows-exe`
- `sleep-noise-guard-android-apk`

## 默认参数

- 噪音阈值：45 dB
- 出现几秒后触发：留空
- 出现几次后触发：留空
- 每次反馈播放次数：留空
- 留空含义：检测到超过阈值的噪音后，立即反馈一次
- 冷却时间：60 秒
- 日志路径：`logs/noise_events.csv`

## 日志格式

Python 桌面版日志默认写入：

```text
logs/noise_events.csv
```

字段包括：

- `timestamp`：噪音出现时间
- `hour`：小时统计桶
- `date`：日期统计桶
- `db`：估算分贝
- `triggered`：是否触发反馈
- `trigger_count`：累计触发次数
- `feedback_repeats`：本次反馈播放次数
- `sounds`：播放的音效文件
